BÁSICO
Verificação de Temperatura
numpy · random.uniform · arrays · condicionais
ENUNCIADO
Uma máquina industrial deve operar entre 60°C e 90°C.

Gere aleatoriamente 10 leituras de temperatura (valores entre 50 e 100°C) e, para cada leitura, informe se está DENTRO ou FORA do limite seguro de operação.

No final, exiba o total de alertas encontrados.
random.uniform() np.array() for loop condicionais if/else np.sum()
DICAS
💡 Dica 1 : Como gerar os dados aleatórios
▼
random.uniform(a, b) gera um decimal aleatório entre a e b. Pense em como repetir essa chamada 10 vezes e guardar os resultados em uma estrutura numpy.

Dica de estrutura: existe uma forma compacta de gerar uma lista de valores em uma linha só, pesquise list comprehension em Python.
💡 Dica 2 : Como verificar os limites
▼
Python permite verificar dois limites ao mesmo tempo em uma única expressão condicional, não precisa de dois if separados.

Pense: como você expressa "está entre X e Y" matematicamente? Tente traduzir isso direto para o código.
💡 Dica 3 : Contar os alertas com numpy
▼
Para contar quantas temperaturas estão fora do limite, use operações vetorizadas do numpy:

alertas = np.sum((temperaturas < TEMP_MIN) | (temperaturas > TEMP_MAX))

O operador | é o "ou lógico" para arrays numpy. O np.sum() conta os True como 1 e False como 0.

random.uniform() while/for + break acumulação de valor np.array() histórico

A cada iteração, três coisas precisam acontecer: gerar o desgaste, atualizar a espessura, e verificar o limite. Em que ordem faz mais sentido?
💡 Dica 2 : Como parar o loop cedo

Existe uma palavra chave em Python que encerra um loop imediatamente quando uma condição é atingida sem esperar o fim das iterações.

Pesquise sobre break e pense em onde exatamente no seu loop ele deve ser colocado.

Análise de Vibração do Motor
np.random.normal · estatística · np.clip · classificação
ENUNCIADO
Simule 100 leituras de vibração de um motor (em mm/s) usando distribuição normal (média=5, desvio padrão=2), com valores limitados entre 0 e 15 mm/s.

Calcule: média, mediana, desvio padrão, mínimo, máximo e o percentual de leituras acima de 10 mm/s (nível crítico).

Classifique o motor: BOM (média < 7) | ATENÇÃO (7–10) | CRÍTICO (> 10).
np.random.normal() np.clip() np.mean() / np.median() / np.std() np.min() / np.max()
DICAS
💡 Dica 1 : Distribuição normal vs uniforme
▼
random.uniform distribui valores uniformemente e todos com mesma chance. Sensores reais se comportam diferente: os valores se concentram em torno de um ponto central.

Pesquise np.random.normal() e seus parâmetros loc, scale e size. O que cada um representa fisicamente para um sensor de vibração?
💡 Dica 2 : Limitando os valores com np.clip()
▼
A distribuição normal pode gerar valores fora dos limites físicos do sensor (negativos ou muito altos). Existe uma função numpy que apara um array entre um mínimo e um máximo.

Pesquise np.clip() e pense: quais são os limites físicos que fazem sentido para este sensor?
💡 Dica 3 : Calculando o percentual crítico
▼
Uma expressão como array > valor no numpy retorna um array de True e False. Você pode somar ou contar esses valores diretamente.

Pense: qual função numpy converte uma contagem em proporção automaticamente?

Plano de Lubrificação
np.zeros · random.randint · múltiplos contadores · lógica temporal
ENUNCIADO
Uma fábrica tem 8 máquinas. Cada máquina deve ser lubrificada a cada 200 horas de operação. Simule 500 horas de operação com incrementos aleatórios de 1 a 5h por turno.

Cada máquina pode ter uma pequena variação de carga (±1h por turno). Registre quando e quantas vezes cada máquina foi lubrificada.
np.zeros() random.randint() np.clip() múltiplos arrays paralelos lógica de acumulação
DICAS
💡 Dica 1 : Arrays paralelos para rastrear múltiplas máquinas
▼
Com 8 máquinas, você precisa rastrear várias informações em paralelo. Pense em quantos arrays você precisa: um para as horas acumuladas de cada máquina, outro para saber quando cada uma foi lubrificada pela última vez, e um terceiro para contar as lubrificações.

np.zeros(n) cria um array de zeros com n posições e um jeito limpo de inicializar todos os contadores de uma vez.
💡 Dica 2 : Loop principal de simulação
▼
O loop principal controla o tempo global da simulação. A cada turno, cada máquina acumula horas ligeiramente diferentes (variação de ±1h).

Uma operação direta entre arrays numpy (array_a + array_b) atualiza todos os valores de uma vez, mais eficiente do que um loop interno. Como você garantiria que os incrementos nunca sejam negativos?
💡 Dica 3 : Verificação de lubrificação
▼
A condição de lubrificação depende do tempo decorrido desde a última lubrificação não do tempo total. Como você calcularia essa diferença usando os arrays que criou?

Quando uma máquina é lubrificada, o que precisa ser atualizado nos seus arrays? Pense nos dois valores que mudam nesse momento.

horas[m] - ultima_lub[m] calcula quantas horas se passaram desde a última lubrificação.

Desgaste de Componentes
random.uniform · simulação · loops · estatísticas
ENUNCIADO
Uma planta tem 50 máquinas. Cada máquina tem um nível de desgaste inicial aleatório entre 0% e 30%. Cada dia, o desgaste aumenta um valor aleatório entre 0,5% e 2,5%. Se o desgaste superar 80%, a máquina entra em falha.

Simule 90 dias de operação e reporte: quantas máquinas falharam, em que dia cada uma falhou, e qual foi a média de vida útil das máquinas que falharam.
random.uniform() dicionários loops aninhados listas de falhas np.mean()
DICAS
💡 Dica 1 : Estrutura de dados: inicializando as máquinas
▼
Você precisa de uma estrutura para guardar o desgaste de cada máquina e outra para registrar quando cada uma falhou. Pense: uma lista indexada serve para os desgastes, mas qual estrutura associa uma máquina ao dia em que falhou?

Para as máquinas ainda ativas, considere uma estrutura que permita remover elementos facilmente durante o loop, sem desorganizar os índices.
💡 Dica 2 : Simulando os 90 dias com loops aninhados
▼
A simulação usa dois loops aninhados: o externo percorre os 90 dias, o interno percorre apenas as máquinas ainda em operação.

Atenção: não modifique um conjunto enquanto itera sobre ele. Colete as falhas do dia em uma lista separada e remova-as do conjunto depois que o loop interno terminar.
💡 Dica 3 : Calculando e reportando os resultados
▼
Os dados de que você precisa já estão no dicionário de falhas. Como extrair apenas os dias (valores) para calcular a média de vida útil?

Para exibir as falhas em ordem cronológica, pesquise como usar sorted() com o parâmetro key= em um dicionário. Lembre-se também de tratar o caso em que nenhuma máquina falhou.

Priorização de Ordens
índice de prioridade · ordenação · listas · avance ponderado
ENUNCIADO
Um técnico recebe 20 ordens de trabalho no início da semana. Cada ordem tem uma criticidade (1–10), um tempo estimado em horas (0,5h a 8h) e um percentual de avanço já concluído (0% a 60%).

Calcule o índice de prioridade com a fórmula:

prioridade = criticidade × (1 − avanco) / tempo_estimado

Ordene as ordens, identifique quais entram no turno do dia (8h disponíveis), tomando as de maior prioridade até esgotar o tempo, e calcule o avanço médio ponderado do turno.
random.uniform() / randint() sorted() com key= acumulador de horas média ponderada list comprehension
DICAS
💡 Dica 1 : Gerando e estruturando as ordens
▼
Cada ordem de trabalho tem vários atributos (criticidade, tempo, avanço). Em Python, um dicionário é ideal para agrupar esses campos e uma lista de dicionários representa o conjunto de todas as ordens.

Pense em como gerar os valores aleatórios para cada atributo usando as funções corretas: randint para inteiros e uniform para decimais.
💡 Dica 2 : Calculando a prioridade e ordenando
▼
Você pode adicionar uma nova chave a cada dicionário da lista percorrendo-a com um loop, basta aplicar a fórmula diretamente.

Para ordenar uma lista de dicionários por um campo específico, pesquise sorted() com o parâmetro key=. Como você faria para que a maior prioridade venha primeiro?
💡 Dica 3 : Selecionando ordens do turno e calculando o avanço ponderado
▼
Para montar o turno, percorra as ordens ordenadas e vá acumulando horas. A condição de entrada é simples: a ordem cabe se o total acumulado não ultrapassar 8h.

Para o avanço médio ponderado: cada ordem contribui proporcionalmente ao seu tempo. Pense na fórmula de média ponderada (é a soma de (valor × peso) dividida pela soma dos pesos).
